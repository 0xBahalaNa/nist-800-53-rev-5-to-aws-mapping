# NIST 800-53 Rev 5 to AWS Service Mapping

| Control ID | AWS Service | Description | FedRAMP High | CJIS Delta |
|------------|-------------|-------------|:------------:|------------|
| AU-2 | AWS CloudTrail | CloudTrail enables audit event logging by capturing API calls across AWS services. Organizations configure trail settings to define which management and data events are logged, satisfying the requirement to identify events the system is capable of auditing. | true | N/A |
| AU-3 | AWS CloudTrail | CloudTrail log entries contain event name, timestamp, source IP, user identity (ARN), request parameters, and response elements. This satisfies the requirement for audit records containing what type of event occurred, when, where, the source, the outcome, and the identity of associated individuals or subjects. | true | N/A |
| AU-12 | AWS CloudTrail | CloudTrail provides system-wide audit record generation capability. When configured as an organization trail via AWS Organizations, it generates audit records for all member accounts across all AWS regions, capturing both management events and data events. FedRAMP High requires multi-region coverage — a single-region trail does not satisfy this control. | true | N/A |
| CM-3 | AWS CloudTrail | CloudTrail provides the audit trail component of configuration change control by recording who made each change, when, from which source IP, and what API action was called. Combined with AWS Config's before-and-after configuration snapshots, CloudTrail satisfies the requirement to document configuration change decisions and retain records of configuration-controlled changes. | true | N/A |
| AC-2 | AWS IAM | IAM provides account management capabilities including creating, enabling, modifying, disabling, and removing accounts. IAM policies enforce conditions on account usage, and integration with AWS Organizations enables centralized account management across multiple AWS accounts. | true | CJIS v6.0 requires accounts with access to CJI be reviewed at minimum quarterly. This exceeds the FedRAMP High baseline review frequency and requires documented evidence of each review cycle with explicit reauthorization or removal of access. |
| AC-3 | AWS IAM | IAM enforces approved authorizations for logical access through a default-deny policy evaluation chain. Every API call is evaluated against identity-based policies, resource-based policies, permissions boundaries, and service control policies (SCPs). Access is denied unless an explicit allow exists and no explicit deny overrides it. | true | N/A |
| AC-6 | AWS IAM | IAM enforces least privilege through identity-based policies, resource-based policies, permissions boundaries, and service control policies (SCPs). IAM Access Analyzer identifies resources shared with external entities and validates policies against least-privilege best practices. | true | N/A |
| IA-2 | AWS IAM | IAM authenticates users through passwords, access keys, and MFA devices. IAM Identity Center provides centralized SSO with support for SAML 2.0 and OIDC federation, enabling multi-factor authentication for all organizational users accessing AWS resources. | true | CJIS v6.0 requires AAL2 multi-factor authentication for all users accessing CJI. This exceeds the FedRAMP High baseline by mandating phishing-resistant authenticators (hardware tokens or push-based MFA) and prohibiting SMS-based OTP as a second factor for CJI systems. |
| IA-5 | AWS IAM | IAM enforces authenticator management through configurable password policies (minimum length, complexity, rotation period, reuse prevention) and access key rotation tracking. IAM credential reports provide visibility into password and access key age across all users, supporting the requirement to manage system authenticator lifecycle. | true | N/A |
| AC-2 | AWS Organizations | AWS Organizations enables centralized account management across multiple AWS accounts through organizational units (OUs) and SCPs. Accounts can be created, moved between OUs, suspended, and closed centrally, satisfying the requirement for organization-wide account management and enforcement of account usage conditions. | true | N/A |
| AC-17 | AWS IAM Identity Center | IAM Identity Center controls remote access to AWS resources by authenticating users through a centralized portal with MFA enforcement before granting session credentials. Permission sets define what authenticated remote users can access, and session duration limits control how long remote access remains valid. | true | N/A |
| IA-2 | AWS IAM Identity Center | IAM Identity Center provides centralized authentication with built-in MFA support including TOTP, hardware security keys (FIDO2), and push notification authenticators. It federates with external SAML 2.0 and OIDC identity providers, enabling organizations to extend their existing identity infrastructure to AWS access. | true | CJIS v6.0 requires AAL2 multi-factor authentication for all users accessing CJI. This exceeds the FedRAMP High baseline by mandating phishing-resistant authenticators (hardware tokens or push-based MFA) and prohibiting SMS-based OTP as a second factor for CJI systems. |
| IA-8 | AWS IAM Identity Center | IAM Identity Center supports federation with external identity providers via SAML 2.0 and OIDC, enabling identification and authentication of non-organizational users from trusted partner organizations. Attribute-based access control (ABAC) can enforce authorization decisions based on identity attributes asserted by the external IdP. | true | N/A |
| AC-17 | AWS Client VPN | AWS Client VPN establishes encrypted TLS tunnels for remote access to VPC resources. It enforces authentication through mutual TLS certificates, Active Directory credentials, or SAML-based federation. Authorization rules control which network CIDR ranges authenticated users can reach, and connection logging provides audit trails of all remote access sessions. | true | N/A |
| IA-5 | AWS Secrets Manager | Secrets Manager manages authenticator lifecycle by storing credentials encrypted with KMS, enforcing automatic rotation on configurable schedules, and providing versioned secret history. Lambda-based rotation functions update credentials in both Secrets Manager and the target service atomically, satisfying the requirement to manage system authenticator content changes. | true | N/A |
| IA-8 | Amazon Cognito | Cognito uniquely identifies and authenticates non-organizational users through user pools with configurable sign-up and sign-in flows. It supports federation with external identity providers (social, SAML, OIDC), adaptive authentication with risk-based challenges, and MFA enforcement, satisfying the requirement to identify and authenticate non-organizational users. | true | N/A |
| AU-2 | Amazon VPC Flow Logs | VPC Flow Logs capture network-level events that CloudTrail does not — accepted and rejected traffic at the ENI, subnet, or VPC level. This supplements CloudTrail's API-layer audit logging with network-layer event logging, satisfying the requirement to identify events the system is capable of auditing across both application and network tiers. | true | N/A |
| AU-6 | Amazon CloudWatch Logs Insights | CloudWatch Logs Insights provides the query and analysis capability needed to review audit records. Analysts can write structured queries against CloudTrail logs and VPC Flow Logs to identify unauthorized access attempts, privilege escalation patterns, and anomalous activity, satisfying the requirement to review and analyze system audit records for indications of inappropriate or unusual activity. | true | CJIS v6.0 Section 5.4 requires audit logs capturing CJI access be retained for a minimum of one year (365 days), reviewed at minimum weekly, and monitored using automated tools to flag suspicious activity. Organizations must assign personnel to review logs and demonstrate a documented review process, exceeding the FedRAMP High baseline review frequency. |
| AU-6 | Amazon Athena | Athena enables SQL-based analysis of CloudTrail logs stored in S3, supporting audit record review over historical data beyond CloudWatch's default retention window. This is particularly valuable for long-term trend analysis and incident forensics, satisfying the requirement to review and analyze audit records for indications of inappropriate or unusual activity. | true | N/A |
| AU-9 | Amazon S3 | S3 protects audit information from unauthorized modification and deletion through multiple mechanisms: S3 Object Lock (WORM compliance mode) prevents log objects from being deleted or overwritten for a defined retention period, bucket policies with explicit deny statements block delete operations even from privileged users, and server-side encryption with KMS-managed keys protects audit records at rest. | true | N/A |
| SC-28 | Amazon S3 | S3 protects information at rest using server-side encryption with KMS-managed keys (SSE-KMS). When configured with a customer managed key (CMK), the organization retains full control over key material, key rotation policy, and key access grants. Default bucket encryption can be enforced via bucket policies that deny PutObject requests lacking SSE-KMS headers. | true | CJIS v6.0 requires agency-managed customer managed keys (CMK) for encrypting CJI at rest — AWS-managed keys do not satisfy this requirement. If AWS-managed keys are used, AWS personnel have theoretical access to decrypt CJI, which brings them into CJIS personnel security scope and requires fingerprint-based background checks for cloud provider staff. Agency-managed CMKs keep key material outside AWS control, removing cloud provider personnel from CJIS personnel security scope and making the deployment viable. |
| AU-6.1 | AWS Security Hub | Security Hub provides automated audit record review and analysis by continuously evaluating findings from CloudTrail, Config, and GuardDuty against security standards (CIS Benchmarks, NIST 800-53, PCI DSS). Automated findings surface misconfigurations and anomalies without requiring manual log review, satisfying the AU-6(1) enhancement for automated integration of audit review, analysis, and reporting. Required at FedRAMP High baseline. | true | N/A |
| SC-7 | Amazon VPC | VPC provides boundary protection through multiple layers: security groups act as stateful firewalls at the instance level controlling inbound and outbound traffic by protocol, port, and source/destination; network ACLs provide stateless filtering at the subnet level; route tables control traffic flow between subnets and to internet/NAT gateways; and VPC peering and transit gateways control inter-VPC communication. This layered architecture satisfies the requirement to monitor and control communications at external and key internal boundaries. | true | N/A |
| CM-7 | Amazon VPC | VPC security groups and network ACLs enforce least functionality at the network layer by restricting which ports, protocols, and source/destination addresses are permitted. Security groups operate on a default-deny-inbound model — only explicitly allowed traffic reaches the resource. Combined with AWS Config Rules like restricted-ssh and restricted-common-ports, unauthorized port exposure is detected and flagged, satisfying the requirement to prohibit or restrict the use of specified functions, ports, protocols, and services. | true | N/A |
| SC-7 | AWS Network Firewall | AWS Network Firewall provides managed stateful inspection at VPC ingress and egress points, enabling deep packet inspection, protocol-based filtering, and IDS/IPS capabilities using Suricata-compatible rules. It supplements VPC security groups and NACLs with application-layer visibility, satisfying the requirement for boundary protection mechanisms that deny traffic by default and allow by exception at managed interfaces. | true | N/A |
| SC-8 | AWS Certificate Manager | ACM provisions TLS certificates that enable encryption of data in transit across AWS services. Certificates deployed to ALB, CloudFront, and API Gateway terminate TLS connections, ensuring transmission confidentiality. ACM automates certificate renewal, preventing lapses in encryption coverage due to expired certificates. | true | N/A |
| SC-8 | Amazon CloudFront | CloudFront enforces transmission confidentiality through configurable TLS security policies that set the minimum TLS version (TLS 1.2 required for FedRAMP) and cipher suites for viewer-facing and origin-facing connections. The viewer protocol policy can require HTTPS-only, redirecting HTTP requests to HTTPS. Origin protocol policies enforce TLS between CloudFront and backend services. | true | N/A |
| SC-12 | AWS KMS | KMS establishes and manages cryptographic keys through a centralized service backed by FIPS 140-2/3 validated HSMs. Organizations create customer managed keys (CMKs) with configurable key policies, automatic annual rotation, and CloudTrail-logged key usage. Key policies and IAM grants control which principals can use, manage, and administer keys, satisfying the requirement to establish and manage cryptographic keys in accordance with organizational requirements. | true | N/A |
| SC-13 | AWS KMS | KMS provides NIST-approved cryptographic protection using AES-256-GCM for symmetric encryption and elliptic curve or RSA algorithms for asymmetric operations, all executed within FIPS 140-2 Level 2 validated HSMs (Level 3 in AWS GovCloud). In GovCloud regions, all KMS API endpoints use FIPS 140-3 validated TLS endpoints, satisfying the requirement to implement FIPS-validated cryptography for the protection of information. | true | N/A |
| SC-28.1 | AWS KMS | KMS customer managed keys (CMKs) provide the cryptographic mechanism for SC-28(1), enabling organizations to implement encryption of information at rest with organization-controlled key material. CMKs integrate with S3, EBS, RDS, and other AWS storage services to encrypt data with keys that the organization creates, rotates, disables, and deletes — not AWS. Required at FedRAMP High baseline. | true | N/A |
| SC-28 | Amazon EBS | EBS encrypts data at rest on volumes using AES-256 encryption with KMS-managed keys. Account-level default encryption can be enabled to automatically encrypt all new EBS volumes and snapshots without requiring per-volume configuration, satisfying the requirement to protect the confidentiality and integrity of information at rest. | true | N/A |
| SC-28 | Amazon RDS | RDS encrypts data at rest for DB instances, automated backups, read replicas, and snapshots using AES-256 encryption with KMS-managed keys. Encryption must be enabled at instance creation and applies to the underlying storage, automated backups, and snapshots, satisfying the requirement to protect the confidentiality and integrity of information at rest. | true | N/A |
| CM-2 | AWS Config | AWS Config records the configuration state of AWS resources as configuration items, creating a point-in-time snapshot each time a resource is created, modified, or deleted. The configuration timeline provides a historical record of how each resource's configuration has changed over time, satisfying the requirement to develop, document, and maintain a current baseline configuration of the system. | true | N/A |
| CM-3 | AWS Config | AWS Config captures the before-and-after configuration state of resources when changes occur, providing the 'what changed' component of configuration change control. Config delivers configuration change notifications via SNS, enabling automated review of changes. Combined with CloudTrail's record of who initiated each change, Config satisfies the requirement to determine, document, and review configuration-controlled changes. | true | N/A |
| CM-6 | AWS Config | AWS Config Rules evaluate whether resource configurations comply with desired settings. AWS managed rules provide pre-built checks (s3-bucket-server-side-encryption-enabled, iam-password-policy, restricted-ssh), and custom rules use Lambda functions for organization-specific checks. Resources are marked COMPLIANT or NON_COMPLIANT continuously, and Config can trigger SSM Automation runbooks for automated remediation, satisfying the requirement to establish, implement, and monitor configuration settings. | true | N/A |
| CM-8 | AWS Config | AWS Config automatically discovers and inventories AWS resources across supported resource types (EC2 instances, S3 buckets, IAM roles, security groups, and 380+ others). Config Advanced Queries enable SQL-like queries against the inventory (e.g., 'SELECT resourceId WHERE resourceType = AWS::EC2::Instance AND configuration.encrypted = false'). The resource inventory updates automatically as resources are created, modified, or deleted, satisfying the requirement to develop and document a system component inventory that accurately reflects the system. | true | N/A |
| CM-8 | AWS Systems Manager | SSM Inventory collects OS-level metadata from managed instances including installed applications, OS version, network configuration, Windows updates, and running services. This extends AWS Config's cloud-layer inventory with instance-layer detail that Config cannot see, satisfying the requirement for a component inventory at the level of granularity deemed necessary for tracking and reporting. | true | N/A |
